  SMS WEATHER AUTO-RESPONDER — Setup Notes
  Tasker Project for Android
  Data Source: NAIPS (Airservices Australia)

OVERVIEW
--------
This Tasker project automatically replies to incoming SMS messages
containing an ICAO airport code (e.g. YSSY, YBDV, YMML) with the
current METAR and TAF for that location, pulled from NAIPS.

HOW IT WORKS (Technical)
  The script uses the NAIPS SOAP Web Service — a proper API that
  Airservices Australia exposes at:
    https://www.airservicesaustralia.com/naips/briefing-service

  Each request carries your credentials in the SOAP XML body.
  No browser login, no cookies, no session management needed.
  A single HTTP POST returns the briefing data directly.

  This is the same API used by tools like Get-AirServicesBriefing
  (PowerShell) and other aviation automation projects.


FILES INCLUDED
--------------
1. SMS_Weather_Responder.prj.xml  — Tasker project (import this)
2. weather_fetch_naips.js         — JavaScript for JavaScriptlet (NAIPS SOAP)
3. weather_fetch.js               — Alternative using aviationweather.gov (no auth, works for major airports)
4. naips_weather_fetch.js         — Old NAIPS web scraping version (deprecated, kept for reference)
5. Setup_Notes.txt                — This file


REQUIREMENTS
------------
• Tasker (v5.0+ recommended, tested on 6.x)
• Active internet connection (mobile data or WiFi)
• NAIPS account (free — register at https://www.airservicesaustralia.com/naips/Account/Register)
• SMS permissions granted to Tasker


QUICK SETUP
============

STEP 1: Import the Tasker Project
  1. Copy SMS_Weather_Responder.prj.xml to your phone
  2. In Tasker: long-press home icon (bottom bar) → Import Project
  3. Navigate to the file and import

STEP 2: Paste the JavaScript
  1. Open the imported project
  2. Go to Tasks → "Fetch Weather and Reply"
  3. Tap Action 3 (JavaScriptlet)
  4. Replace the placeholder code with the ENTIRE contents of
     weather_fetch_naips.js
  5. Change these two lines at the top:
       var NAIPS_USER = "YOUR_USERNAME_HERE";
       var NAIPS_PASS = "YOUR_PASSWORD_HERE";
     to your actual NAIPS username and password.
     IMPORTANT: Username MUST be UPPERCASE.

STEP 3: Test
  From another phone, send an SMS to your number:
    YBDV
  You should receive a reply with METAR and TAF.

  If you get "NAIPS login failed" → double-check credentials.
  If you get no reply → check Tasker is running and has SMS permissions.


MANUAL BUILD (if XML import fails)
====================================

STEP 1: Create the Task
  Tasker → Tasks tab → + → Name: "Fetch Weather and Reply"

  Action 1 — Variable Set
    Name:  %sms_body
    To:    %SMSRB

  Action 2 — Variable Set
    Name:  %sms_from
    To:    %SMSRF

  Action 3 — JavaScriptlet
    Code:  (paste the ENTIRE contents of weather_fetch_naips.js)
    Timeout: 45 seconds
    Auto Exit: unchecked

  Action 4 — If
    Condition: %valid  eq  1

  Action 5 — Send SMS
    Number:  %sms_from
    Message: %reply

  Action 6 — End If

STEP 2: Create the Profile
  Profiles tab → + → Event → Phone → Received Text
  Type: Any
  Leave Sender and Content fields empty
  Back → select task "Fetch Weather and Reply"

HOW THE SOAP REQUEST WORKS
===========================
The script sends a SOAP XML request like this:

  POST https://www.airservicesaustralia.com/naips/briefing-service
  Content-Type: text/xml
  SOAPAction: ...briefing-service?wsdl

  <loc-brief-rqs password="xxx" requestor="XXX" source="sms-wx">
    <loc>YBDV</loc>
    <flags met="true"/>
  </loc-brief-rqs>

The response contains a <content> element with plain text
including METAR, TAF, ATIS, NOTAM etc. The script extracts
just the METAR and TAF lines.

The "loc-brief" operation supports up to 12 locations per
request. Currently the script sends one at a time (matching
the SMS use case).


CUSTOMISATION
=============

Change reply format:
  Edit the JavaScript — look for the "reply +=" lines near the bottom.

Only respond to specific senders:
  In Profile → Event (Received Text) → set Sender field.

Add a keyword trigger (e.g. "WX YBDV"):
  Change the regex in the JavaScript:
    var m = body.match(/^WX\s+([A-Z]{4})$/);

Request multiple products:
  To include ATIS, change the flags line in the SOAP XML:
    '<ns1:flags met="true" ntm="false"/>'
  Then add an ATIS extraction regex similar to METAR/TAF.

Include NOTAMs:
  Set ntm="true" in the flags. Add parsing for NOTAM blocks.

Switch to aviationweather.gov (for major airports only):
  Replace the JavaScriptlet code with weather_fetch.js.
  No credentials needed, but only covers international airports.


TROUBLESHOOTING
===============

No reply received:
  - Check Tasker is active (green dot in notification bar)
  - Check SMS and Internet permissions for Tasker
  - Add a Flash action after the JavaScriptlet for debugging:
    Text: valid=%valid icao=%icao

"NAIPS login failed" reply:
  - Username must be UPPERCASE
  - Password is 7-20 alphanumeric characters only
  - Try logging into NAIPS website to verify credentials work

Empty reply / "N/A" for both:
  - The station may not have current METAR/TAF issued
  - Try a known station like YSSY or YMML to verify connectivity

JavaScriptlet error in Tasker log:
  - Ensure code is pasted exactly (no extra line breaks at start)
  - Set Tasker Preferences → Misc → JavaScript Engine to "Auto"

Slow response:
  - The NAIPS SOAP service typically responds in 1-3 seconds
  - The 45-second timeout prevents hangs on poor connections
